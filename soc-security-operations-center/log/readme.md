This folder contains my notes and write-up on **Log Fundamentals**, covering how security logs are generated, structured, and used to detect suspicious activity.

## Overview

Logs are the raw evidence trail behind almost every security investigation — every login, process execution, and network connection leaves a trace. This room covers the fundamentals of reading and interpreting that trail before moving into more advanced correlation with tools like a SIEM.

## Writeup

📄 `log-fundamentals.md` — Covers core log types (authentication, system, application, network), log structure, and how to identify suspicious activity by reviewing raw log data.

## SOC Analyst Relevance

Log analysis is the foundation of nearly every SOC task — from triaging a single alert to reconstructing a full incident timeline, the ability to read and interpret logs directly is a core, everyday skill for an analyst.
