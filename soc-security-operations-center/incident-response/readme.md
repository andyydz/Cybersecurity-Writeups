This folder contains my notes and write-up on **Incident Response Fundamentals**, covering how security incidents are identified, contained, and resolved once a threat has been confirmed.

## Overview

Incident response picks up where detection leaves off — once an alert or investigation confirms malicious activity, this is the structured process a SOC follows to contain the threat, eliminate it, recover normal operations, and learn from what happened.

## Writeup

📄 `insidence-responce-fundamentals.md` — Covers the core phases of incident response (detection, containment, eradication, recovery, post-incident analysis), and how established frameworks like SANS and NIST structure this process.

## SOC Analyst Relevance

Incident response is the natural next step after alert triage — understanding these phases gives context for how a confirmed alert escalates into a full response, and what an analyst's role looks like at each stage.
